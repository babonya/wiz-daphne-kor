======================================================================
Wizardry Daphne Dungeon Macro Script - v18.11.1 Release
======================================================================

본 스크립트는 모바일 게임 '위저드리 다프네'의 던전 루프 파밍을 
자동화하기 위한 파이썬 기반의 이미지 매칭 매크로 솔루션입니다.

(★중요 안내: 현재 이 매크로는 '백야 1층' 파밍 후 복귀(회군)하는 루트로 전용 설계되어 있습니다. 타 던전에 대한 호환 및 추가 기능 구현 여부는 향후 업데이트 계획이 불투명(미정)합니다.)

----------------------------------------------------------------------
1. 에뮬레이터 (MuMu Player) 권장 세팅
----------------------------------------------------------------------
* 해상도 설정:
  - 무조건 [태블릿/사용자 지정 해상도] -> [2560 * 1440] (360 DPI)으로 세팅해주셔야 합니다.
  - (★부연 설명: 제작자가 이 매크로를 처음 개발할 당시 MuMu 플레이어의 기본 세팅 환경을 그대로 기반으로 삼아 좌표를 설계했기 때문에 해당 해상도로 고정되어 있습니다.)
  - 게임 내 세팅이나 해상도가 해당 스펙과 다르면 이미지 템플릿 검출 좌표가 왜곡되어 정상 작동하지 않습니다.
* 디버그 모드 (ADB) 활성화:
  - MuMu 플레이어 설정에서 디버깅 모드(ADB)를 항상 허용으로 설정해 주셔야 파이썬이 통신할 수 있습니다.
  - 기본 접속 포트는 통상 16384 (MuMu 플레이어 12 기본값)이며, 스크립트 실행 시 자동으로 포트를 스캔하여 연결합니다.

----------------------------------------------------------------------
2. 파이썬 환경 구축 및 필수 라이브러리 설치
----------------------------------------------------------------------
* 파이썬 설치:
  - Python 3.10 버전 이상의 윈도우용 파이썬을 설치합니다.
  - 설치 시 반드시 [Add python.exe to PATH] 옵션을 체크하여 환경 변수에 등록해 주세요.
* 필수 라이브러리 설치:
  - 명령 프롬프트(cmd)를 열고 아래 명령어를 입력하여 필요한 라이브러리를 전수 설치합니다.
    pip install opencv-python pillow numpy pure-python-adb

----------------------------------------------------------------------
3. main.py 사용자 환경 변수 커스텀 설정
----------------------------------------------------------------------
`main.py` 최상단 [글로벌 제어 세팅 변수 구역]의 아래 설정들을 본인의 계정 환경에 맞춰 직접 변경해 사용할 수 있습니다.

* LIMIT_DUNGEON_LOOPS = 2
  - 던전을 몇 바퀴 주회하고 마을(여관)로 회군하여 정비 및 부활/레벨업 시퀀스를 밟을지 횟수를 설정합니다.
* START_RUN_COUNT_OFFSET = 2
  - 최초 실행 시 초기 카운트를 몇으로 시작할지 오프셋을 지정합니다. (바로 마을 회군을 테스트하려면 이 값을 LIMIT_DUNGEON_LOOPS와 같게 설정하면 유용합니다.)
* ENABLE_FIRST_COMBAT_SKILL = False
  - (★주의: 사용 금지, 현재 에러로 인해 미구현된 기능입니다. 반드시 False로 유지하여 사용하세요.)
  - 첫 전투 시 숏컷 기반의 광역기 및 버프 스킬 예약을 사용할지 여부를 제어합니다.

----------------------------------------------------------------------
4. 힐러 및 따개 캐릭터 템플릿(도장) 등록 방법 (매우 중요)
----------------------------------------------------------------------
유저 캐릭터의 구성에 맞게 `templates` 폴더 내에 힐러와 따개(함정 해제 캐릭터)의 도장 이미지를 캡처하여 저장해두어야 매크로가 이를 식별하고 자동으로 캐릭터 스왑 및 힐을 진행합니다.

* 따개(함정 해제) 캐릭터 등록:
  - 함정 상자를 열었을 때 나오는 캐릭터 선택 화면에서, 함정을 해제할 캐릭터(예: 밀라나 등)의 이름/얼굴 영역을 캡처하여 templates 폴더에 png 파일로 저장합니다.
  - 파일명 명명 규칙: `disarmer_[우선순위]_[이름].png`
  - 예시: `disarmer_1_milana.png`
  - 앞자리 번호의 알파벳/숫자 정렬 순으로 화면에서 매칭 우선순위를 가집니다.
  - (★주의: 현재 상자 개봉 미니게임은 타이밍 바를 실시간 계산하는 대신 '무지성 고속 연타 난타' 방식으로 해제를 유도합니다. 따라서 함정 해제 확률이 높은 전용 캐릭터(민첩 등 함정 해제 수치가 높은 캐릭터)를 따개 이미지로 등록하여 사용하길 강력히 권장합니다.)

* 힐러(일괄 치료 진입용) 캐릭터 등록:
  - 파티창에 진입했을 때, 힐러 캐릭터(예: 예카테리나, 앨리스 등)의 얼굴 마크를 캡처하여 templates 폴더에 png 파일로 저장합니다.
  - 파일명 명명 규칙: `healer_[우선순위]_[이름].png`
  - 예시: `healer_1_ekaterina.png`, `healer_2_alice.png`
  - 파티에 복수의 힐러가 동시 존재하더라도 앞번호 힐러를 우선적으로 탐색하여 정비창에 진입하므로, 마나 소모 분배 등을 유연하게 통제할 수 있습니다.

----------------------------------------------------------------------
5. 매크로 실행 방법
----------------------------------------------------------------------
* MuMu 플레이어로 게임을 켜고 던전 필드 내에 캐릭터를 위치시킵니다.
* 배포판 폴더에 동봉된 `run.bat` 배치 파일을 더블클릭하면 자동으로 에뮬레이터를 찾아 통신을 연결하고 주행을 개시합니다.
* (★자동전투 사전 팁):
  - 매크로 프로그램 수준에서의 상세 전투 스킬 제어는 아직 완벽히 구현되지 않았습니다.
  - 따라서 매크로를 본격적으로 실행하기 전, 수동 전투로 1~2바퀴 정도 직접 던전을 주행하며 인게임 자체의 자동전투 스킬 발동 조건을 적절히 정돈한 상태에서 매크로를 가동하시길 권장합니다.
* 로그 보관 및 용량 관리:
  - 매크로 실행 시마다 `logs/` 폴더 내에 실행 로그 파일이 자동으로 백업 누적됩니다.
  - 디스크 공간을 확보하기 위해 오래된 로그 파일들은 주기적으로 수동으로 지워주시는 것을 권장합니다.
* 에러 발생 시 자가 조치 방법:
  - 주행 도중 예기치 못한 에러로 매크로가 멈추거나 튕기는 경우, 에러 발생 시점의 인게임 화면 캡처 이미지와 `logs/` 폴더에 생성된 해당 시각의 로그 텍스트를 함께 확보하세요.
  - 이 정보들을 챙겨서 Gemini의 Antigravity AI 어시스턴트에 첨부 및 전달하고 개인 맞춤형 수정을 요청하시면, AI가 버그를 읽고 즉각 코드를 자가 디버깅하여 해결책을 도출해 드립니다.

======================================================================
Wizardry Daphne Dungeon Macro Script
======================================================================

본 스크립트는 모바일 게임 '위저드리 다프네'의 던전 루프 파밍을 
자동화하기 위한 파이썬 기반의 이미지 매칭 매크로 솔루션입니다.

----------------------------------------------------------------------
기능
----------------------------------------------------------------------
1. 백야에서 지정된 횟수의 주회 이후 마을에 복귀해서 여관 숙박 (추후 다른 던전 추가 예정)
2. 지정된 횟수의 전투 이후 자동 힐 (기본 세팅: 매 전투 후 자동 힐)
3. 아군/주인공 빈사(딸피) 시 자동 힐링 (v1.14.0에서 픽스되었으나 아직 검증되지 않음)
4. 앱/뮤뮤 프리징 시 앱/뮤뮤 자동 재부팅 (재부팅 후 자동으로 평타 주회 모드 가동)

* 던전 작동 범위:
  - 현재 버전은 "백아의 동굴 지하 1층" 구역을 순회하고 마을로 복귀하도록 특화되어 있습니다.
  - 전투 중 캐릭터가 사망하거나 파티가 빈사 상태(부활 불가, HP 소진 등)가 될 경우 화면이 정상 템플릿과 매칭되지 않아 매크로가 멈출 수 있습니다. 이 현상은 v1.14.0에서 임계값 자동 완화 및 치료 연동 가드를 적용하여 픽스하였으나, 아직 구체적인 구동 검증은 완료되지 않았으므로 오작동 시 제보 바랍니다.

----------------------------------------------------------------------
1. 에뮬레이터 (MuMu Player) 및 게임 내 설정 권장 세팅
----------------------------------------------------------------------
* 에뮬레이터 해상도 설정:
  - MuMu Player의 기본 표준 해상도인 [2560 * 1440] (360 DPI, 가로/세로 비율) 설정을 변경 없이 그대로 유지해주셔야 합니다.
  - 해당 세팅은 매크로 이미지 템플릿이 이 기본 해상도 스펙을 기준으로 제작되었기 때문입니다.
  - 해상도를 임의로 변경할 경우 이미지 템플릿 검출 좌표가 왜곡되어 정상 작동하지 않습니다.
* 에뮬레이터 디버그 모드 (ADB) 활성화:
  - MuMu 플레이어 설정에서 디버깅 모드(ADB)를 항상 허용으로 설정해 주셔야 파이썬이 통신할 수 있습니다.
  - 기본 접속 포트는 통상 16384 (MuMu 플레이어 12 기본값)이며, 스크립트 실행 시 자동으로 포트를 스캔하여 연결합니다.

* 위저드리 다프네 게임 내 인터페이스(UI) 및 인게임 설정:
  1. 던전 내부 이동 시 미니맵 아래 영역을 '확장'하여 4개의 단추 아이콘이 화면에 모두 노출된 상태여야 합니다.
     (노출되어야 하는 4개 아이콘: 이전이동 반복, 던전나가기, 체크포인트 이동, 보물함 이동)
     아이콘이 가려져 있거나 접혀 있으면 매크로가 단추를 매칭하지 못해 오작동하거나 렉 복구 시퀀스가 기동하지 않습니다.
  2. 던전 내 미니맵과 화면 상태는 반드시 게임 초기 상태(기본 줌 세팅) 그대로 유지해야 합니다.
     사용자가 인위적으로 맵을 줌인(확대) 또는 줌아웃(축소)한 상태일 경우, 맵의 크기나 비율이 달라져 매크로의 이동 제어 및 매칭 엔진이 정상 작동하지 않습니다.
  3. 인게임 [설정] (톱니바퀴 아이콘) 세팅 권장 가이드:
     - 화질 설정: [속도우선]
     - 프레임 제한: [30fps]
     - 자동선택: [OFF]
     - 텍스트 일괄 표시: [ON]
     - 밝기 설정: [최소]

2. 파이썬 환경 구축 및 필수 라이브러리 설치
----------------------------------------------------------------------
* 파이썬 설치:
  - Python 3.10 버전 이상의 윈도우용 파이썬을 설치합니다.
  - 설치 시 반드시 [Add python.exe to PATH] 옵션을 체크하여 환경 변수에 등록해 주세요.
* 필수 라이브러리 설치:
  - 명령 프롬프트(cmd)를 열고 아래 명령어를 입력하여 필요한 라이브러리를 전수 설치합니다.
    pip install opencv-python pillow numpy pure-python-adb

----------------------------------------------------------------------
3. main.py 사용자 환경 변수 커스텀 설정
----------------------------------------------------------------------
`main.py` 최상단 [글로벌 제어 세팅 변수 구역]의 아래 설정들을 본인의 계정 환경에 맞춰 직접 변경해 사용할 수 있습니다.

* LIMIT_DUNGEON_LOOPS = 2
  - 던전을 몇 바퀴 주회하고 마을(여관)로 회군하여 정비 및 부활/레벨업 시퀀스를 밟을지 횟수를 설정합니다.
* START_RUN_COUNT_OFFSET = 2
  - 최초 실행 시 초기 카운트를 몇으로 시작할지 오프셋을 지정합니다. (바로 마을 회군을 테스트하려면 이 값을 LIMIT_DUNGEON_LOOPS와 같게 설정하면 유용합니다.)
* ENABLE_FIRST_COMBAT_SKILL = False
  - 첫 전투 시 숏컷 기반의 광역기 및 버프 스킬 예약을 사용할지 여부를 제어합니다 (True: 수동 예약 모드 / False: 고속 완전 자동전투).
* 에뮬레이터 자동 재부팅 설정 (프리징 감지 자동 On/Off):
  - 앱/에뮬 프리징 및 렉으로 인한 락 발생 시 자동으로 에뮬레이터를 리부트하고 복구하는 기능이 탑재되어 있습니다. 이 기능이 정상적으로 가동되려면 main.py 최상단 제어판의 에뮬레이터 실행 파일 경로(EMU_PATH)를 본인의 PC 환경(MuMu Player 설치 경로)에 맞게 직접 정확하게 설정해 주셔야 합니다.

----------------------------------------------------------------------
4. 힐러 및 따개 캐릭터 템플릿(도장) 등록 방법 (매우 중요)
----------------------------------------------------------------------
유저 캐릭터의 구성에 맞게 `templates/!!Character` 폴더 내에 힐러와 따개(함정 해제 캐릭터)의 도장 이미지를 캡처하여 저장해두어야 매크로가 이를 식별하고 자동으로 캐릭터 스왑 및 힐을 진행합니다.

* 따개(함정 해제) 캐릭터 등록:
  - 함정 상자를 열었을 때 나오는 캐릭터 선택 화면에서, 함정을 해제할 캐릭터(예: 밀라나 등)의 이름 영역을 캡처하여 `templates/!!Character` 폴더에 png 파일로 저장합니다.
  - 파일명 명명 규칙: `disarmer_[우선순위]_[이름].png`
  - 예시: `disarmer_1_milana.png`
  - 앞자리 우선순위 번호 순으로 화면에서 매칭 정렬 순위를 가집니다 (자연 정렬 도입으로 1 ➔ 2 ➔ 10 순으로 정렬됨).

* 힐러(일괄 치료 진입용) 캐릭터 등록:
  - 필드에 진입했을 때, 힐러 캐릭터(예: 예카테리나, 앨리스 등)의 이름을 캡처하여 `templates/!!Character` 폴더에 png 파일로 저장합니다.
  - 파일명 명명 규칙: `healer_[우선순위]_[이름].png`
  - 예시: `healer_1_ekaterina.png`, `healer_2_alice.png`
  - 주인공 빈사 도장 등록 (매우 중요):
    주인공이 빈사 상태일 때 필드에서 나타나는 캐릭터 이름 도장(예: `healer_99_skelton_red.png`)을 직접 캡처하여 templates/!!Character 폴더에 필수로 넣어주어야 합니다. 빈사 상태에서는 붉은 장막 효과로 인해 일반 도장의 이미지 인식이 불가능해집니다. 따라서 빈사 전용 도장(`_red` 접미사 필수 포함)이 등록되어 있어야 봇이 빈사를 정상 감지하고, 유연하게 필드 앵커 임계값을 낮춰 안전하게 고정좌표 힐 시퀀스를 개시합니다.
  - 평상시 힐러 선택 스왑 루프에서 방해가 되지 않도록 빈사 전용 도장은 우선순위 넘버링을 99 등 후순위로 설정하는 것이 권장됩니다.
  - 파티에 복수의 힐러가 동시 존재하더라도 앞번호 힐러를 우선적으로 탐색하여 정비창에 진입하므로, 마나 소모 분배 등을 유연하게 통제할 수 있습니다.

----------------------------------------------------------------------
5. 매크로 실행 방법
----------------------------------------------------------------------
* MuMu 플레이어로 게임을 켜고 던전 필드 내에 캐릭터를 위치시킵니다.
* 배포판 폴더에 동봉된 `run.bat` 배치 파일을 더블클릭하면 자동으로 에뮬레이터를 찾아 통신을 연결하고 주행을 개시합니다.

----------------------------------------------------------------------
6. 로그 관리 및 오류 문의
----------------------------------------------------------------------
* 오래된 로그 정리:
  - 매크로를 실행하면 `logs/` 디렉토리에 실행 이력에 대한 상세 텍스트 파일이 지속적으로 생성되고 백업됩니다.
  - 디스크 공간 절약을 위해 매크로 기동 시 4일이 경과한 오래된 로그 파일들은 자동으로 영구 삭제 처리됩니다.

* 에러 발생 시 대처법:
  - 매크로 오작동 또는 에러가 발생하여 멈추는 경우, 에러가 발생한 당시의 에뮬레이터 화면을 캡처하고, `logs` 폴더의 최신 로그 텍스트 파일 내용을 함께 첨부하여 Gemini의 Antigravity 에이전트에 문의하시면 개인 환경에 맞게 수정을 지원받으실 수 있습니다.

----------------------------------------------------------------------
7. 책임 한계 고지
----------------------------------------------------------------------
* 본 매크로 프로그램을 사용하는 모든 책임은 사용자 본인(매크로 사용자)에게 있습니다. 비정상적인 이용으로 인한 불이익이나 계정 제재 등의 모든 책임은 사용자에게 귀속됩니다.
